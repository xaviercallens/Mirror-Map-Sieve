## Downloading Weights

```bash
# Install huggingface-hub
pip install huggingface-hub

# Download all models
python -c "
from huggingface_hub import snapshot_download
models = [
    'SocrateAI/deepproblog-mnist',
    'SocrateAI/deepproblog-type-classifier',
    'SocrateAI/deepproblog-step-type',
    'SocrateAI/deepproblog-validity',
    'SocrateAI/deepproblog-vagueness',
]
for m in models:
    snapshot_download(m, local_dir=f'weights/{m.split(\"/\")[-1]}')
"
```

## Training Instructions

### Digit Recognition

The `net_digit` network is a standard LeNet-5 trained end-to-end
via DeepProbLog on the MNIST addition task.

```bash
# Prerequisites
pip install deepproblog problog torch torchvision

# Train
python -m deepproblog.train \
    --program ../programs/digit_recognition.pl \
    --network net_digit:lenet5 \
    --dataset mnist \
    --epochs 30 \
    --lr 1e-3 \
    --batch_size 64 \
    --output_dir weights/net_digit/

# Expected accuracy: ~98% on single digits, ~96% on addition
```

### Type Constraint

The type/shape/device classifiers are 3-layer MLPs trained on
synthetic tensor descriptor embeddings.

```bash
# Generate synthetic training data
python scripts/generate_type_data.py \
    --num_samples 50000 \
    --output data/type_train.json

# Train all three networks jointly
python -m deepproblog.train \
    --program ../programs/type_constraint.pl \
    --network net_type:mlp3,net_shape:mlp3,net_device:mlp2 \
    --dataset data/type_train.json \
    --epochs 50 \
    --lr 5e-4 \
    --output_dir weights/type_nets/
```

### Proof Audit

The proof-audit networks require a dataset of annotated proof steps.
We use a combination of:
- Lean 4 Mathlib proof traces (automatically extracted)
- Synthetic invalid proofs (generated by LLM perturbation)
- Human-annotated vagueness labels

```bash
# Extract proof traces from Mathlib
python scripts/extract_proof_traces.py \
    --mathlib_dir ~/.elan/toolchains/leanprover-lean4-v4.14.0/lib/lean4/ \
    --output data/proof_traces.json

# Generate negative examples
python scripts/generate_invalid_proofs.py \
    --input data/proof_traces.json \
    --output data/invalid_proofs.json

# Train
python -m deepproblog.train \
    --program ../programs/proof_audit.pl \
    --network net_step_type:transformer_small,net_validity:cross_encoder,net_vagueness:bert_tiny \
    --dataset data/proof_traces.json,data/invalid_proofs.json \
    --epochs 100 \
    --lr 2e-5 \
    --output_dir weights/proof_nets/
```

## Model Architecture Details

### LeNet-5 (net_digit)

```
Input: 28×28×1
Conv2d(1, 6, 5) → ReLU → MaxPool(2)
Conv2d(6, 16, 5) → ReLU → MaxPool(2)
Flatten → Linear(256, 120) → ReLU
Linear(120, 84) → ReLU
Linear(84, 10) → Softmax
Parameters: ~62K
```

### MLP-3 (net_type, net_shape)

```
Input: d_embed (128 or 256)
Linear(d_embed, 256) → ReLU → Dropout(0.1)
Linear(256, 128) → ReLU → Dropout(0.1)
Linear(128, n_classes) → Softmax
Parameters: ~100K
```

### Transformer-Small (net_step_type)

```
Input: token sequence (max 512 tokens)
Embedding(vocab_size, 128)
TransformerEncoder(n_layers=2, n_heads=4, d_model=128, d_ff=256)
[CLS] pooling → Linear(128, n_step_types) → Softmax
Parameters: ~500K
```
